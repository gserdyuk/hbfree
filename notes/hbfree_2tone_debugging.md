# Hbfree — проблемы 2-тонального режима

**Назначение:** рабочий документ для отладки 2-tone harmonic balance.
**Аудитория:** автор (Gena) + Claude Code как ассистент.
**Статус:** живой документ, дополняется по мере расследования.
**Версия:** 1.1 (обновлено после первой сессии диагностики)

**Текущие симптомы:**
- частая плохая сходимость;
- при сходимости — расхождение 10–20% с другими симуляторами (Tina, time-domain + FFT);
- специфика: усечение спектра, свободная (плавающая) фаза тонов;
- **наблюдаемо:** якобиан часто вырождается, направление шага Ньютона и градиент невязки почти перпендикулярны.

## 0. Как пользоваться этим документом

Документ построен как **чеклист возможных причин**, сгруппированных по слоям:
спектр → матрица преобразования → якобиан → ньютон → начальное приближение → численная точность → сравнение с референсом.

Для каждой причины указаны:
- **симптом** — как проявляется;
- **где смотреть** — место в коде / конфиге;
- **как проверить** — минимальный эксперимент;
- **как чинить** — типовое решение;
- **статус** — открыта / закрыта / подтверждена.

Отладку вести **сверху вниз**: ошибка в слое N делает диагностику слоя N+1 бессмысленной.

Закрытые разделы оставлены в документе с явной пометкой, чтобы:
- не возвращаться к ним повторно;
- иметь контекст, если при новых находках понадобится пересмотр.

## 1. Спектр: выбор и усечение набора гармоник

### 1.1 Схема усечения: box vs diamond

**Статус:** частично закрыта. Используется **diamond**, задаётся явно парами `(m, n)`.

**Что остаётся проверить:** значение предельного порядка `K` (условие `|m|+|n| ≤ K`) для типичных задач. Если `K = 3..4` — может теряться часть IM-продуктов высокого порядка и давать вклад в 10–20%.

**Эксперимент на будущее.** Для конкретного тестового случая, где наблюдается 10–20% расхождение, прогнать с `K = 3, 5, 7`. Если результат стабилизируется к `K = 5` — дефолт поднять до 5–6. Если монотонно «ползёт» до `K = 7` — причина не в спектре.

**Оригинальная справка.** Для 2-tone HB гармоники индексируются парой `(m, n)`, частота `f_{m,n} = m·f1 + n·f2`:
- **box:** `|m| ≤ M, |n| ≤ N` — `(2M+1)(2N+1)` точек;
- **diamond:** `|m| + |n| ≤ K` — `2K² + 2K + 1` точек;
- **advanced table:** только продукты до заданного порядка интермодуляции.

### 1.2 Отрицательные частоты и эрмитова симметрия

**Статус:** открыта, низкий приоритет. Если бы тут был баг — расхождение было бы не 10–20%, а качественное (неправильная форма сигнала).

Проверяется тривиальным тестом: transform → inverse transform в обход солвера должен давать исходный сигнал с машинной точностью.

### 1.3 DC-компонента

**Статус:** открыта, низкий приоритет.

Проверка: чистая DC-задача (амплитуды тонов в нуль) должна совпасть с отдельным DC-солвером.

## 2. Частотные соотношения тонов

### 2.1 Соизмеримость f1 и f2 (commensurate case)

**Статус:** открыта, проверить при первом же «не сошлось». Если `f1/f2 = p/q` с малыми `p, q` (например 1 GHz и 2 GHz), часть пар `(m, n)` совпадает по частоте → вырождение.

Быстрый тест: заменить на `0.9 GHz / 1.1 GHz`. Если сходимость радикально лучше — обработка соизмеримых случаев нужна.

### 2.2 Малое отношение тонов (close tones)

**Статус:** открыта, средний приоритет.

Для двумерной DFT (которая используется) близкие тоны менее проблемны, чем для APFT, но проверить стоит.

### 2.3 Свободная (плавающая) фаза тонов — **КЛЮЧЕВАЯ ГИПОТЕЗА**

**Статус:** **ОСНОВНОЙ ПОДОЗРЕВАЕМЫЙ.**

**Почему это главный кандидат на причину вырождения якобиана:**

Автономная система по каждому возбуждаемому тону имеет фазовую симметрию: решение определено с точностью до произвольного сдвига фазы. Математически это означает, что якобиан **обязан** иметь нулевое собственное значение по направлению этой симметрии — по одному нулю на каждый тон без фиксированной фазы.

Следствия:
- якобиан сингулярен → шаг Ньютона ложится в нуль-пространство или близкую к нему область;
- градиент невязки «смотрит» поперёк — в направления, не покрываемые этим вырождением;
- **результат: шаг почти перпендикулярен градиенту — в точности наблюдаемый симптом.**

Без явного фазового условия сходимость либо невозможна, либо случайна (зависит от того, какая мода численно «проснётся» первой).

**Как проверить:** см. §11 — SVD якобиана.

**Как чинить:** для каждого тона с неопределённой фазой добавить одно вещественное уравнение:

```
Im(V_{входной_узел_тона}^{(m,n)=тон}) = 0
```

(или аналог: фиксация мнимой части на любой компоненте, однозначно принадлежащей тону). Соответственно — строка и столбец в якобиане. Система становится квадратной и невырожденной. Goldstein-Armijo продолжает работать как есть.

**NB.** Если тон возбуждается внешним источником с заданной фазой — фазовое условие для этого тона не нужно (фаза закреплена источником). Нужно только для автономных колебаний и для тонов с неопределённой опорной фазой.

## 3. Матрица преобразования время ↔ частота

### 3.1 Выбор узлов времени

**Статус:** **ЗАКРЫТА.** Используется двумерное DFT с прямым произведением времён `(t1, t2)`. Обусловленность такой матрицы близка к идеальной (для равномерной сетки — числа обусловленности порядка 1). Эта ветвь подозрений снята.

**Справка (на будущее, если понадобится).** Три типовых подхода:
- **2D прямое произведение** ← используется сейчас;
- **APFT** — одномерный набор точек с подбором шага; чувствителен к качеству подбора;
- **MFT / Rizzoli** — детерминированное построение.

### 3.2 Оверсэмплинг

**Статус:** **открыта.** Точное значение не помнится, надо посмотреть в коде.

Для нелинейности порядка `K` нужен оверсэмплинг **минимум в `K` раз** по каждому из двух времён. Для типового MOSFET в слабой нелинейности (кубика в Тейлоре) — `×3`, для сильной нелинейности — `×5`.

**Как проверить:** прогнать тест с разным оверсэмплингом (множители 2, 3, 5). Если результат меняется — aliasing есть, нужно поднимать.

**Как чинить.** Увеличить число временных точек по каждому измерению сетки. Для 2D DFT стоимость растёт как `N1·N2`, терпимо.

### 3.3 Нормировка DFT

**Статус:** открыта, низкий приоритет. Ошибка в нормировке даёт **систематический множитель** на всех амплитудах (неверные фазы исключены). Если бы это был источник 10–20% — расхождение было бы ровно в 0.5, 1/√2 или 2 раза, а не в диапазоне. Скорее всего, нормировка корректна.

## 4. Якобиан

### 4.1 Аналитический vs численный

**Статус:** **ЗАКРЫТА.** Якобиан аналитический. Это снимает самый опасный класс проблем со сходимостью.

### 4.2 Производные моделей транзисторов

**Статус:** открыта, средний приоритет.

Даже при аналитическом якобиане в целом, отдельные модели транзисторов могут содержать баги в производных `∂I/∂V`, `∂Q/∂V`. Эти баги не ловятся DC-анализом (где используются только сами `I(V)`), но убивают HB.

**Связь с планами по Hbfree:** это прямо соотносится с планом «добавить тесты для основных нелинейных моделей». **Тесты должны покрывать не только I(V), но и все частные производные** — сверка аналитики с конечной разностью в 5–10 характерных точках режима работы.

### 4.3 Якобиан линейной части (Y-матрица)

**Статус:** открыта, низкий приоритет (после закрытия §4.1).

Проверка: линейная схема без нелинейностей в HB должна точно совпадать с независимым AC-анализом на каждой частоте спектра.

### 4.4 Вырождение якобиана — наблюдаемый симптом

**Статус:** **наблюдается, требует целенаправленной диагностики.** См. §11.

Краткий итог анализа трёх возможных причин вырождения:

| Причина | Статус | Комментарий |
|---|---|---|
| Недоопределённость из-за свободной фазы | **ОСНОВНОЙ ПОДОЗРЕВАЕМЫЙ** | См. §2.3 и §11 |
| Физическое вырождение линейной части | **ЗАКРЫТА** | По оценке автора — исключено |
| Численное вырождение из-за масштабирования | **ЗАКРЫТА** | Масштабирование в коде предусмотрено |

**Вывод:** с вероятностью, близкой к единице, вырождение вызвано отсутствием фазового условия. Экспериментальное подтверждение — §11.

## 5. Ньютон и стратегия сходимости

### 5.1 Начальное приближение

**Статус:** **подтверждённая проблема — source stepping отсутствует.** Нужна доработка.

**Текущее состояние:** начальное приближение, по всей видимости, нулевое или линейное. Для HB с сильной нелинейностью этого недостаточно.

**Что есть vs что нужно:**
- есть: Goldstein-Armijo line search (контроль шага в окрестности);
- нет: source stepping / continuation по амплитуде возбуждения (подвод к решению издалека).

Это **разные механизмы**. G-A работает, когда Ньютон уже «видит» решение и нужно не перескочить. Source stepping нужен, когда начальная точка далеко, и линеаризация в ней не указывает на решение вообще.

**Реализация:** начать с амплитуд тонов × 0.01 от целевых, сойтись (быстро, почти линейный режим), использовать как старт для × 0.1, далее × 0.5, × 1.0. Типично 3–5 ступеней, каждая — одна сходимость Ньютона.

**Приоритет: высокий.** Это второй после фазового условия источник «часто не сходится».

### 5.2 Damping / line search

**Статус:** **ЗАКРЫТА (есть).** Goldstein-Armijo реализован.

### 5.3 Критерий сходимости

**Статус:** частично проработано, нужно уточнить поведение.

**Текущие значения:**
- абсолютный tol = `1e-12`;
- относительный tol ≈ `1e-6`.

**Что стоит добавить в логирование.** При выходе из ньютоновского цикла писать:
- финальная `||r||`;
- финальная `||r|| / ||r_0||`;
- какой из критериев сработал (абсолютный / относительный / лимит итераций).

Если почти всегда срабатывает относительный при `1e-6`, для плохо обусловленных 2-tone задач это может ограничивать реальную точность — и давать часть «процентной» ошибки.

**Возможное решение:** сделать tol_rel конфигурируемым, при подозрениях уметь ужесточить до `1e-8`.

### 5.4 Максимальное число итераций

**Статус:** открыта, низкий приоритет.

Важно логировать причину выхода (сходимость vs лимит итераций). Если цикл часто выходит по лимиту — это **не успех**, а замаскированный провал.

## 6. Численная точность

### 6.1 Точность real/complex

**Статус:** **ЗАКРЫТА** по факту (для gfortran — 8 байт восстановлены после PR Дитмара; IFX использует свои флаги).

**Почему это важно для 2-tone:**
- DFT-матрица для умеренных спектров имеет `cond ~ 10^3..10^5`;
- каждое решение системы теряет `log10(cond)` значащих цифр;
- при 4-байтной точности (7 цифр) остаётся 2–4 — недостаточно для `tol = 1e-8..1e-12`;
- при 8-байтной (15–16) остаётся 10–13 — норма.

**Действие:** добавить комментарий в мейкфайле рядом с флагами:
```
# -r8 / -i8 required for 2-tone HB convergence and precision.
# Do not remove. See hbfree_2tone_debugging.md §6.1.
```

### 6.2 Масштабирование

**Статус:** **ЗАКРЫТА.** Масштабирование в коде предусмотрено.

## 7. Сравнение с референсом (Tina, time + FFT)

### 7.1 Методология

Без изменений от версии 1.0. Ключевые требования:
1. одинаковый набор частот при сравнении;
2. одинаковые или согласованные фазы тонов;
3. одинаковые модели на обеих сторонах;
4. time-domain должен быть сошедшимся к стационарному режиму — прогон длиной в несколько периодов самой низкой разностной частоты как минимум.

### 7.2 Шкала расхождения

| Расхождение | Вероятная причина |
|---|---|
| < 0.1% | норма, численный шум |
| 0.1–1% | разная точность моделей, оверсэмплинг |
| 1–5% | недостаточный спектр, слабый оверсэмплинг |
| 5–20% | **текущий случай** — с учётом новой картины: недо­сошедшийся Ньютон из-за вырожденного якобиана + возможно слабый оверсэмплинг |
| > 20% | структурная ошибка |

### 7.3 Изолированные тесты

Последовательность от простого к сложному:
1. линейная схема 2-tone → точное совпадение с AC на каждой частоте;
2. слабая квадратичная нелинейность → сверка с теорией возмущений;
3. IM3 slope: два тона равной амплитуды, IM3 на `2f1 − f2`, наклон `A³` на log-log;
4. сильная нелинейность → только сверка с time-domain.

## 9. Что добавить в регрессионные тесты

- unit-тесты DFT 2-tone: прямое+обратное = тождество;
- unit-тесты производных моделей: dI/dV (аналитика vs конечная разность) в 5–10 точках режима;
- интеграционный: линейный 2-tone vs AC;
- интеграционный: квадратичная нелинейность vs теория возмущений;
- интеграционный: IM3 slope = 3 ± 0.05 на log-log;
- тест воспроизводимости: вход → выход с точностью `1e-10`.

## 11. Диагностический эксперимент — SVD якобиана (**приоритет 1**)

**Цель:** однозначно подтвердить или опровергнуть фазовую свободу как причину вырождения якобиана.

### 11.1 Подготовка

Выбрать **один** тестовый случай, в котором воспроизводимо наблюдается симптом «шаг ⊥ градиент» или вырождение. Желательно простой: 2 нелинейных каскада, 2 тона без искусственно закреплённой фазы.

### 11.2 Инструментация кода

В месте сборки якобиана перед решением линейной системы — одноразовый дамп в файл. Пример (Fortran-псевдокод):

```fortran
if (debug_dump_jacobian .and. iter .eq. dump_at_iter) then
  open(unit=99, file='jacobian_dump.txt', status='replace')
  do i = 1, n
    do j = 1, n
      write(99, '(2E25.16)') real(J(i,j)), aimag(J(i,j))
    end do
  end do
  close(99)
  open(unit=98, file='residual_dump.txt', status='replace')
  do i = 1, n
    write(98, '(2E25.16)') real(r(i)), aimag(r(i))
  end do
  close(98)
end if
```

Параметры `debug_dump_jacobian` и `dump_at_iter` — из конфига. Дамп на первой-второй итерации, когда эффект уже виден.

### 11.3 Анализ (отдельный скрипт, Python)

```python
import numpy as np

n = <размер системы>
J = np.zeros((n, n), dtype=complex)
with open('jacobian_dump.txt') as f:
    for i in range(n):
        for j in range(n):
            re, im = map(float, f.readline().split())
            J[i, j] = re + 1j * im

U, S, Vh = np.linalg.svd(J)

print(f"sigma_max = {S[0]:.3e}")
print(f"sigma_min = {S[-1]:.3e}")
print(f"cond(J)   = {S[0]/S[-1]:.3e}")
print()
print("10 smallest singular values:")
for k, s in enumerate(S[-10:]):
    print(f"  sigma_{n-10+k} = {s:.3e}")

print()
print("Right singular vectors for smallest sigmas:")
for k in range(3):
    v = Vh[-(k+1), :]
    idx = np.argsort(np.abs(v))[::-1][:5]
    print(f"  vector {k} (sigma={S[-(k+1)]:.3e}):")
    for i in idx:
        print(f"    component[{i}] = {v[i]:.3e}  (|.|={abs(v[i]):.3e})")
```

### 11.4 Ожидаемые результаты и их интерпретация

**Сценарий A (подтверждение фазовой свободы):**
- `sigma_min ~ 1e-14..1e-16` (машинный ноль);
- число таких «нулей» равно числу тонов без фазового закрепления (для 2-tone обычно 2);
- правые сингулярные векторы малых σ локализованы на компонентах, соответствующих мнимой части коэффициентов при парах `(1, 0)` и `(0, 1)` на входных узлах.

**Действие:** реализовать фазовое условие (§2.3, блок «как чинить»).

**Сценарий B (есть малые σ, но вектора не соответствуют фазам):**
- есть дополнительный источник вырождения;
- смотреть, на каких переменных сосредоточен вектор — это прямо указывает на место бага.

**Действие:** разобрать индивидуально, вернуться к §4.3 или §2.1.

**Сценарий C (малых σ нет, cond умеренная):**
- якобиан не вырожден;
- «перпендикулярность шага и градиента» — артефакт чего-то другого: возможно некорректный выбор итерации для диагностики, или особенность записи в логе.

**Действие:** пересмотреть, как именно измеряется перпендикулярность. Возможно, нужны другие метрики диагностики.
