## Именованные константы (PARAMETER)

### funcsize.i

| Константа | Значение | Смысл |
|-----------|----------|-------|
| `MAXU_MD` | 10 | макс. напряжений на модель |
| `MAXD_MD` | 10 | макс. производных на модель |
| `MAXKN`   | 20 | макс. частот (базовый) |
| `MAXKN1`  | 200 = 10 × MAXKN | расширенный для производных |
| `B1_SIZE` | 256 | FFT-буфер |
| `B2_SIZE` | 4096 = 16 × B1_SIZE | разреженный 2D FFT-вектор |

### main.f

| Константа | Значение | Смысл |
|-----------|----------|-------|
| `ISIZE_MAXNODE` | 200 | узлов в схеме до редукции |
| `ISIZE_MAXVAR`  | 1000 | переменных (узлы × частоты после редукции) |

## Дерево зависимостей размеров

```
MAXKN = 20
│
├─ MAXKN1 = 10 × MAXKN = 200
│    └── KR1(200), W1(200), MN1(2,200), NNR1(20)    [/BLK1/, /BLW1/]
│
├─ KR(20), W(20), WR(20,20), WS(20,20), MN(2,20)    [/BLK1/, /BLW1/, /BLW2/]
│
└─ S(200, 20)  ← второе измерение = MAXKN            [main.f]


MAXU_MD = 10, MAXD_MD = 10
│
├─ znef.f:   UNEL( MAXU_MD × MAXKN )             = 200
├─ znef.f:   ZNN(  (MAXU_MD+MAXD_MD) × MAXKN )   = 400
├─ znejac.f: UNEL( MAXU_MD × MAXKN1 )            = 2000
└─ znejac.f: ZNN1( (MAXU_MD+MAXD_MD) × MAXKN1 )  = 4000


ISIZE_MAXNODE = 200
│
├─ YY(200, 200)          — матрица проводимостей       [main.f]
├─ S(200, 20)            — (20 = MAXKN)                [main.f]
├─ NOUZ(200), VECTJ(200), INOUZ1(200)                  [main.f]
│
└─ ISIZE_MAXVAR = 1000   — предположительно:
                           ≈ (MAXNODE после редукции) × MAXKN
                           но явно не выражена через другие константы


B1_SIZE = 256
└─ B2_SIZE = 16 × 256 = 4096                           [znef.f, znejac.f]


Libelem (все 37 файлов):
└─ Y(15,15), J(15), SV(15,15)  — макс. 15 узлов на элемент
                                  (не объявлено как PARAMETER — "магическое число")
```

## Проблемные места

### 1. Коллизия значений: `ISIZE_MAXNODE = 200` и `MAXKN1 = 200`
Одинаковое значение, разная семантика: узлы схемы vs. частоты×производные.  
Риск ошибочной подстановки при рефакторинге.

### 2. `ISIZE_MAXVAR = 1000` — "магическая" константа
Не выражена через другие константы. По комментарию в коде: `узлы_после_редукции × MAXKN`.  
Коэффициент редукции нигде явно не зафиксирован.

### 3. `500` в `/POINT/` и `/NODEL/` — "магические числа"
Не объявлены как `PARAMETER`, не связаны с другими константами явно.

### 4. `6000` в `BUFFER` — неочевидное происхождение
Возможно `ISIZE_MAXVAR × 6` = 1000×6, или `ISIZE_MAXNODE × 30`.  
Не документировано.

### 5. `15` в libelem — продублировано в 37 файлах
Максимальное количество узлов на элемент нигде не объявлено как `PARAMETER`.  
При необходимости изменить — придётся редактировать все 37 файлов вручную.

### 6. `/SAVEDATA_CB/` — крупнейший блок, размеры зашиты вручную
`YEL_SAVE(2,20,1000)`, `VJ_SAVE(2,20,200)`, `NOY_SAVE(2,20,2000,2)` — ~100K DCOMPLEX.  
При увеличении `MAXKN` или `ISIZE_MAXNODE` нужно обновлять вручную.
